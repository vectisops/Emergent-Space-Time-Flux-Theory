## 🌌 1. The Theory: What is ESFT?
Emergent Spacetime Flux Theory (ESFT) proposes a fundamental shift in how we understand the fabric of the universe. In standard general relativity, spacetime is treated as a foundational, continuous geometric backdrop—a static stage on which matter and energy interact. 

ESFT argues that spacetime is not fundamental. Instead, it is an **emergent structure** generated by the active coherence of underlying, localized flux dynamics. 

Imagine looking at a digital photograph. From a distance, you see a smooth, continuous image (spacetime). But if you zoom in, the image is actually constructed of individual pixels turning on and off (the flux). In ESFT, gravity and spatial geometry are not just the "bending" of a static grid; they are measurable density variations within this foundational flux.

## 🔭 2. How it Fits: The 98% Problem
The mainstream consensus of the Standard Cosmological Model asserts that roughly 98% of the universe is composed of "Dark Matter" and "Dark Energy." These are substances that are completely invisible, practically unmeasurable by standard physical means, and interact with nothing in our realm. 

ESFT rejects this assumption. When a scientific framework requires 98% of the universe to be made of invisible, undetectable material just to balance the equations, it is highly probable that the underlying geometric assumptions are flawed.

Instead of inventing "dark" variables to explain why galaxies spin too fast or why the universe expands, ESFT addresses the root framework. Those observed anomalies are not caused by invisible matter—they are measurable, localized variations in the spacetime flux itself. ESFT aims to test physical realities, not mathematical patches.

## 📡 3. Planned Experiments
Because ESFT relies on real-world measurements rather than invisible constants, our experimental wing is heavily focused on physical telemetry and localized anomaly detection.

* **High-Density Spatial Mapping:** Utilizing custom multi-rotor UAS (Unmanned Aerial Systems) flying automated grid patterns to capture micro-variations in gravitational and spatial telemetry over diverse topographies.
* **Electromagnetic Flux Auditing:** Deploying software-defined radio (SDR) and HackRF platforms to map localized electromagnetic anomalies and correlate them with spatial variance, testing the boundary where EM fields and spacetime flux interact.
* **Offline AI Hypothesis Generation:** Feeding raw telemetry and hardware data into locally hosted, air-gapped Large Language Models (via Ollama) to cross-reference anomalies against ESFT topologies in real-time.

## 🚀 Deployment & Setup

This architecture requires unrestricted local model processing and hardware-level USB access for SDR tools. It is designed to run via Native Windows (PowerShell) or Windows Subsystem for Linux (WSL2).

**1. Clone the repository:**
```powershell
git clone [https://github.com/yourusername/esft-core.git](https://github.com/yourusername/esft-core.git)
cd esft-core
```

**2. Configure the Local Environment:**
To establish local directories, Python virtual environments, and offline storage limits, execute the setup script.

*For Native Windows:*
```powershell
.\src\local_environment.ps1
```
*For WSL2 / Linux:*
```bash
bash src/local_environment.sh
```

**3. Initialize the Offline AI Orchestrator:**
Ensure the [Ollama Windows App](https://ollama.com/download/windows) (or your WSL instance) is running in the background. Build the custom analytical agent using the ESFT framework instructions:
```powershell
ollama create esft-analyst -f orchestration/Modelfile
```

**Note on Hardware Telemetry:** If executing on Windows and using HackRF/SDR auditing tools, it is highly recommended to run the telemetry modules via WSL2 using `usbipd-win` to ensure native USB passthrough for frequency captures.
